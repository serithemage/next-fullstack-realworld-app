# ADR-010: 미들웨어 체인 패턴

## Status

승인 (Accepted)

## Context

Next.js의 미들웨어는 `middleware.ts`에서 단일 함수로 내보내야 하지만, 프로젝트에는 세 가지 독립적인 미들웨어 관심사가 있다:
1. **피드 라우팅**: 인증 상태에 따른 홈페이지 피드 리다이렉트
2. **인증 보호**: `/editor`, `/settings` 경로 접근 제어
3. **국제화**: locale 감지 및 라우팅

이 세 관심사를 단일 미들웨어 함수에 모두 넣으면 코드가 비대해지고 유지보수가 어렵다.

고려한 대안:
- **단일 미들웨어에 모두 작성**: 단순하지만 관심사 분리 없음, 수정 시 전체에 영향
- **matcher 기반 분리**: Next.js `config.matcher`로 경로별 분리하지만 미들웨어 함수는 여전히 하나
- **서드파티 라이브러리**: `next-middleware-chain` 등이 존재하지만 불필요한 의존성 추가

## Decision

**함수형 합성(functional composition) 기반 미들웨어 체인 패턴을 직접 구현한다.**

```
chain([withFeed, withAuth, withIntl])
```

- `src/middlewares/chain.ts`에 `chain()` 함수 구현
- 각 미들웨어는 `(middleware: NextMiddleware) => NextMiddleware` 시그니처의 팩토리 함수
- 재귀적으로 합성하여 실행 순서: withFeed → withAuth → withIntl

## Consequences

**긍정적:**
- 각 미들웨어가 독립 파일로 분리되어 관심사 분리 달성
- 새 미들웨어 추가 시 배열에 추가만 하면 됨
- 외부 의존성 없이 20줄 미만의 코드로 구현
- 실행 순서가 배열 순서로 명확히 표현됨

**부정적:**
- 미들웨어 간 실행 순서에 암묵적 의존성 (withIntl은 반드시 마지막)
- 디버깅 시 체인 내 어느 미들웨어에서 문제가 발생했는지 추적이 어려울 수 있음
- 미들웨어 간 데이터 전달은 request headers나 cookies를 통해야 함
