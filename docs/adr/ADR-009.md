# ADR-009: 소프트 삭제 패턴

## Status

승인 (Accepted) — 2023-11-24

## Context

RealWorld 스펙에서 글과 댓글의 삭제 기능이 필요하다. 삭제된 데이터의 복구 가능성, 데이터 무결성(외래 키 참조), 감사 추적을 고려해야 한다.

고려한 대안:
- **하드 삭제 (물리적 삭제)**: 단순하지만 복구 불가, 외래 키 CASCADE 필요
- **소프트 삭제 + deletedAt 타임스탬프**: 삭제 시점 기록 가능하지만 스키마 복잡
- **별도 아카이브 테이블**: 완전한 분리지만 조회 로직 이중화

## Decision

**`del: Boolean @default(false)` 플래그를 사용한 소프트 삭제 패턴을 채택한다.**

- Article과 Comment 모델에 `del` 필드 추가
- 삭제 시 `del: true`로 업데이트 (물리적 삭제 없음)
- 조회 시 항상 `where: { del: false }` 필터 적용

## Consequences

**긍정적:**
- 데이터 복구가 단순 (`del: false`로 변경)
- 외래 키 참조가 유지됨 (CASCADE 불필요)
- 삭제된 데이터도 관리자 조회/통계에 활용 가능

**부정적:**
- 모든 조회 쿼리에 `del: false` 필터를 빠뜨리지 않도록 주의 필요
- Prisma 미들웨어로 자동 필터링을 적용하지 않아 수동 관리
- 시간이 지나면 삭제된 데이터가 DB에 누적됨 (주기적 정리 필요)
- `deletedAt` 타임스탬프가 없어 삭제 시점 추적 불가
