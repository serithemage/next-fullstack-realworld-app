# ADR-008: CI/CD 전략 (GitHub Actions + Vercel)

## Status

승인 (Accepted)

## Context

프로젝트에 CI/CD 파이프라인이 없는 상태에서, PR 검증과 자동 배포를 구축해야 한다. 배포 대상은 Vercel이며, 코드 품질과 보안을 자동으로 검사하고자 한다.

고려한 대안:
- **Vercel 단독**: 빌드/배포만 가능. 테스트, 보안 스캔 등 커스텀 단계 불가
- **GitHub Actions + Vercel CLI**: 배포를 직접 제어. 세밀하지만 설정 복잡
- **GitLab CI**: GitHub 저장소와 별도 CI 플랫폼 관리 필요

## Decision

**GitHub Actions로 검증 파이프라인을 구축하고, Vercel GitHub Integration으로 배포를 처리한다.**

파이프라인: lint → type-check → build → test(단위/통합 병렬) → E2E → 보안 스캔(npm audit + gitleaks) → Vercel 배포

- PR 생성/업데이트 시: 전체 검증 + Preview 배포
- main 머지 시: 전체 검증 + 프로덕션 배포
- 통합/E2E 테스트는 GitHub Actions의 `services` 블록으로 PostgreSQL 컨테이너 실행

## Consequences

**긍정적:**
- GitHub 네이티브 CI로 PR과 자연스럽게 통합
- Vercel Preview 배포로 리뷰어가 실제 동작 확인 가능
- 보안 스캔(npm audit, gitleaks)으로 취약점/시크릿 노출 사전 차단
- 무료 티어로 충분한 빌드 시간 확보

**부정적:**
- GitHub Actions와 Vercel 배포가 별도로 동작하여 상태 동기화 필요
- PostgreSQL services 컨테이너로 인한 CI 실행 시간 증가
- gitleaks 설정/관리 부담
