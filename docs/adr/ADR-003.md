# ADR-003: NextAuth JWT 인증 전략

## Status

승인 (Accepted)

## Context

RealWorld 스펙은 사용자 등록, 로그인, 인증된 API 호출을 요구한다. Next.js 앱에서 인증을 구현해야 하며, 서버 컴포넌트, API 라우트, 미들웨어 모두에서 인증 상태에 접근해야 한다.

고려한 대안:
- **직접 구현 (커스텀 JWT)**: 완전한 제어권이 있으나 보안 취약점 위험
- **NextAuth + Database 세션**: 세션을 DB에 저장. 안전하지만 매 요청마다 DB 조회 필요
- **Auth.js v5 (beta)**: NextAuth의 차기 버전이지만 안정성 미검증

## Decision

**NextAuth 4 + Credentials Provider + JWT 세션 전략을 사용한다.**

- Credentials Provider로 email/password 기반 인증 구현 (RealWorld 스펙 요구사항)
- JWT 세션 전략으로 DB 조회 없이 빠른 인증 확인
- bcrypt(10 rounds)로 비밀번호 해싱
- 서버: `getServerSession(authOptions)`, 미들웨어: `getToken()`, 클라이언트: `AuthProvider` 컨텍스트

## Consequences

**긍정적:**
- JWT로 미들웨어에서 빠른 인증 확인 가능 (DB 조회 불필요)
- NextAuth의 검증된 보안 패턴 활용
- `callbacks`로 JWT/세션에 사용자 정보(username, id) 주입 가능
- Prisma 어댑터로 사용자 데이터 관리 통합

**부정적:**
- Credentials Provider는 NextAuth에서 "덜 안전한" 옵션으로 분류됨 (OAuth 권장)
- JWT는 서버에서 즉시 무효화 불가 (토큰 만료까지 유효)
- 세션 정보 변경 시 토큰 재발급 필요
