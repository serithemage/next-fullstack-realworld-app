# ADR-005: Tailwind CSS 스타일링 선택

## Status

승인 (Accepted)

## Context

RealWorld 앱의 UI를 구현할 스타일링 솔루션이 필요하다. 컴포넌트 기반 React 앱에서 유지보수가 쉽고 일관된 디자인을 적용할 수 있어야 한다.

고려한 대안:
- **CSS Modules**: Next.js 내장 지원. 스코프된 CSS이지만 유틸리티 부재
- **styled-components**: CSS-in-JS. 동적 스타일에 강하지만 서버 컴포넌트 호환 이슈
- **Sass/SCSS**: 변수/믹스인 지원이지만 유틸리티 퍼스트 접근 아님

## Decision

**Tailwind CSS 3을 사용한다.**

- 유틸리티 퍼스트 접근으로 빠른 프로토타이핑
- PostCSS 기반으로 빌드 시 사용하지 않는 클래스 자동 제거 (작은 번들)
- Next.js 공식 지원 및 통합
- 서버 컴포넌트/클라이언트 컴포넌트 구분 없이 동일하게 사용

## Consequences

**긍정적:**
- 별도 CSS 파일 관리 없이 JSX 내에서 스타일링 완료
- `clsx` 유틸리티와 결합하여 조건부 클래스 적용 간편
- 디자인 토큰(색상, 간격, 폰트)이 설정 파일(`tailwind.config.ts`)에 중앙화
- 빌드 최적화로 프로덕션 CSS 크기 최소화

**부정적:**
- HTML에 클래스가 많아져 가독성 저하
- 복잡한 동적 스타일은 인라인 스타일이나 CSS 변수 필요
- Tailwind 클래스 네이밍을 학습해야 함
