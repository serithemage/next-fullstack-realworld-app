# ADR-002: PostgreSQL + Prisma ORM 선택

## Status

승인 (Accepted) — 2023-11-24

## Context

RealWorld 스펙은 사용자, 글, 댓글, 좋아요, 팔로우 등 관계형 데이터 모델을 필요로 한다. M:N 관계(글-태그, 사용자-좋아요, 사용자-팔로우)가 여러 개 존재하며, 트랜잭션이 필요한 작업(좋아요 토글 시 카운트 동기화 등)이 있다.

고려한 대안:
- **MongoDB + Mongoose**: 스키마 유연성이 높지만 관계형 쿼리에 약함
- **PostgreSQL + TypeORM**: 성숙한 ORM이지만 TypeScript 타입 추론이 약함
- **PostgreSQL + Drizzle**: 경량이지만 생태계가 Prisma 대비 작음

## Decision

**PostgreSQL + Prisma 5를 사용한다.**

- Prisma의 선언적 스키마(`schema.prisma`)로 DB 모델을 코드와 함께 버전 관리
- 자동 생성되는 타입 안전한 클라이언트로 TypeScript와의 통합이 우수
- `prisma migrate dev`로 마이그레이션 관리, `prisma studio`로 DB GUI 제공
- PostgreSQL은 복합키, JSON, 전문 검색 등 풍부한 기능 지원
- Vercel Postgres와의 자연스러운 연동 (Connection Pooling URL 지원)

## Consequences

**긍정적:**
- 스키마 변경이 마이그레이션 파일로 추적됨
- 자동 생성 타입으로 런타임 에러 방지
- `prisma db seed`로 개발용 시드 데이터 관리 편리
- Prisma Client 싱글턴 패턴으로 개발 시 hot-reload 인스턴스 누수 방지

**부정적:**
- Prisma Client 생성 단계 필요 (`postinstall`에서 `prisma generate`)
- Raw SQL이 필요한 복잡한 쿼리에서 ORM 추상화가 제약이 될 수 있음
- 테스트 시 Prisma provider가 DB 종류에 종속 (SQLite ↔ PostgreSQL 호환 불가)
